# 📚 Project: High-Speed Library Inventory PWA

## 0. 운영 맥락 (Context)
**동국대학교사범대학부속가람고등학교** 도서관에서 사서교사 **민경** 선생님이 지도하는 **도서부 동아리 「빛나래」** 및 도서관 업무 과정에서, 동아리 학생들이 장서점검을 수행할 때 사용하는 **모바일 웹 PWA**입니다. UI 문구는 현장 안내(선생님께 확인, 숫자 바코드만 저장 등)를 반영합니다.

## 1. 개요 (Overview)
본 프로젝트는 도서관 장서점검 업무의 병목을 줄이기 위한 **고속 바코드 스캐닝 모바일 웹 PWA**입니다. 브라우저만으로 즉시 구동되며, **점검 세션 단위**로 **연속 스캔(Continuous Scan)** 과 **로컬 즉시 저장**에 맞춰져 있습니다.

## 2. 사용자 스토리 (User Stories)
- **빛나래 동아리원:** “수천 권을 스캔할 때, 점검을 시작한 뒤에는 버튼 없이 바코드만 갖다 대도 자동으로 인식·기록되어야 한다.”
- **빛나래 동아리원:** “앱에 들어오자마자 카메라가 켜지지 않고, **「장서점검 시작」**을 눌렀을 때만 권한·프리뷰가 열렸으면 한다.”
- **빛나래 동아리원:** “한 번 시작한 점검에서 찍은 코드는 **시작 시각을 기준으로 한 저장소 키**에만 줄바꿈으로 쌓이고, **스캔할 때마다 바로** 저장되어야 한다. 중단 시점에 한꺼번에 저장되면 안 된다.”
- **빛나래 동아리원:** “점검을 시작하기 전에 **예전에 저장해 둔 점검 기록**을 목록에서 고르고, 내용을 보거나 고치거나 지울 수 있어야 한다.”
- **빛나래 동아리원:** “QR·문자가 섞인 코드는 장서 번호가 아니므로 **숫자만** 받아들이고, 화면에서 **방금 인식된 값**을 바로 알 수 있어야 한다.”
- **사서교사·도서부:** “서고처럼 네트워크가 불안정해도 스캔 데이터가 **브라우저 로컬에 보존**되고, 이후 보내기·동기화 전략을 붙일 수 있어야 한다.”

## 3. 현재 구현 상태 (Implemented)

### ① 점검 세션 & Continuous Scan
- **명시적 시작:** 진입 시 카메라는 꺼진 상태. **「장서점검 시작」** 시 `beginInventorySession()`으로 세션을 연 뒤에만 카메라 라이프사이클이 시작됨.
- **엔진 선택:** 브라우저가 지원하면 **Native `BarcodeDetector`**를 우선 사용하고, 미지원/초기화 실패 시 **Quagga2**로 폴백.
- **세션 종료:** 진행 화면 상단 "지금까지 점검" 카드 **왼쪽의 셰브런-백 아이콘 버튼**(과거 헤더의 「점검 중단」 텍스트 버튼을 대체) 탭 시 **확인 팝업 없이** 바로 `endInventorySession()` — 카메라·인스턴스 정리. 권수에 따라 aria-label만 "점검 마치고/중단하고 이전 화면으로"로 분기. 이미 스캔분은 스캔 시점마다 저장되어 있으므로 **종료 시 일괄 저장 없음**.
- **종료 후 백업 권유(2026-05-28 추가):** 권수>0으로 종료된 경우 사용자를
  메인이 아니라 **방금 끝난 세션의 지난 점검 상세**로 자동 이동시키고
  (history.replaceState로 scan→detail 교체) 상단에 "사서께 전달하려면 복사
  하세요" 안내 배너를 한 번 노출한다. 복사 버튼을 누르면 배너는 사라진다.
  권수 0(빈 세션)이면 자동 정리 후 종래대로 메인으로 복귀. 모달이 아닌
  비차단형 흐름이라 .cursorrules 9번 원칙("확인 없이 즉시 종료")과 양립.
- **연속 스캔:** 세션 유지 중 별도 촬영 버튼 없이 디코딩 콜백으로 반복 인식.
- **뷰파인더 + ROI 가이드:** 화면 중앙에 **가로 90% × 세로 72%** 가이드 박스를 표시하고, Quagga 폴백 시 비디오 프레임에서 **동일 비율로 중앙 크롭**한 뒤 JPEG로 `decodeSingle`에 넘김(가이드와 픽셀 ROI 일치).
- **Quagga2 폴백 디코딩:** `inputStream.size` **1280**, `locator.patchSize` **`"large"`**, `locator.halfSample` **`false`**(작은 바코드에서 픽셀 뭉개짐 완화). 네이티브 `BarcodeDetector` 경로는 별도 튜닝 없음.
- **카메라 트랙:** `getUserMedia` 직후 지원 기기에서 `applyConstraints`로 해상도·(가능 시) 디지털 줌·연속 초점을 시도하며, 실패 시에도 스트림은 유지.

### ② 검증·피드백·라이브 표시
- **UI 게이트 (`Scanner.tsx`):** `trim` 후 **`/^\d{5,13}$/`** 인 값만 합의·저장 파이프라인으로 전달.
- **스토어 (`appendDigitScanToActiveSession`):** `trim` 후 **`/^\d+$/`** 재검증. **동일 바코드**는 **약 2초** 안의 중복 스캔만 무시(다른 코드는 즉시 허용).
- **멀티 프레임 합의:** 유효 코드가 **2회 연속 동일**할 때만 확정 저장(Native/Quagga 모두 2회).
- **멀티 피드백 (성공 시):**
  - Haptic: `navigator.vibrate([70])` (지원 기기에서만)
  - Visual: 화면 테두리 **녹색 플래시** (`globals.css` `@keyframes scan-success-flash`)
  - UI: **토스트** `기록했어요 · [숫자]` (1.5s 후 자동 닫힘)
  - ※ **청각(비프) 피드백은 사용하지 않음.** 도서관 환경 소음 부담을 이유로
    `useScanBeeps` 훅과 sound-muted 토글을 제거(2026-05-28).
- **실패 안내(비숫자/중복):** 토스트만 사용. 비숫자 코드 인식 시 "숫자가 아니라
  넘어갔어요", 동일 코드 쿨다운(2초) 중복 시 "방금 찍은 번호예요 · NNNN".
- **라이브 인지:** 스캔 중 상단 패널에 **「지금까지 점검」** 권수 + **「방금 인식」** 대형 숫자, 스캔마다 **글로우·스케일** 애니메이션(`scan-live-code-hit`). 하단에는 이번 세션 누적(보기 모드 기본, **직접 수정** 토글로 편집). 보기 모드에서 **`liveSessionText` 갱신 시 textarea 최하단 자동 스크롤**; 수정 모드에서는 중지.
- **기기 정보 접근:** **푸터 인스타 버튼 옆의 `i` 버튼**(`DebugInfoButton`)을 눌러
  브라우저/OS/진동 지원 여부를 다이얼로그로 확인. ESC·바깥 클릭·닫기 버튼 모두
  지원하고 트리거로 포커스 복귀. (과거 점검 진행 화면 상단 우측에 있던 디버그
  트리거는 제거됨.)

### ③ 세션별 localStorage (Offline-First)
- **본문 키 규칙:** `book-scanner:session:` + **점검 시작 시각의 ISO8601 문자열** (예: `book-scanner:session:2026-04-07T12:34:56.789Z`). 한 세션의 모든 스캔은 **동일 키**의 **단일 문자열 값**에 `\n`으로 누적.
- **즉시 저장:** `appendDigitScanToActiveSession`이 매 성공 시 `localStorage` 기존 값을 읽고 **줄 append 후 `setItem`**. 이탈·종료와 무관하게 **스캔 즉시 반영**.
- **세션 메타(2026-05-28 도입, 2026-05-29 휴지통 확장):** 본문과 분리된 네임스페이스 `book-scanner:meta:` + 본문과 동일한 ISO suffix에 JSON `{ backedUpAt: number, deletedAt: number }`로 보관(`src/store/sessionMeta.ts`). 본문 파싱·세션 정리 로직과 완전 분리. 표준 store API: **`markSessionBackedUp(key)`**(복사 성공 시), **`softDeleteSession(key)`**(휴지통으로), **`restoreSession(key)`**(복구). 영구 삭제는 기존 `deleteSessionKey(key)`(본문+메타 동시 제거). 모든 액션은 메타를 `mergeSessionMetaRaw`로 부분 갱신해 다른 필드를 덮어쓰지 않으며, 자동으로 revision을 bump해 목록·휴지통 UI가 즉시 갱신된다. 메인 진입의 빈 세션 정리(`removeSessionKeysWithZeroBarcodes`)는 휴지통 세션을 건드리지 않는다. **meta 부재 = "신규(미복사) + 활성(휴지통 아님)"으로 해석**(마이그레이션 fail-safe). 목록 뱃지: 활성 세션 중 권수>0에 한해 **"복사한 적 있음"**(목련 그린, 완료) 또는 **"신규"**(연꽃 로즈, 강조).
- **Zustand 역할:** 실행 중 UI 메타(`activeSessionKey`, `liveSessionText`, `lastCapturedCode` 등)만 관리. 스캔 화면 노출은 페이지의 **`isScanMode`** 등으로 구분. **스캔 본문은 `persist` 미사용** — 원본은 항상 해당 세션 키의 `localStorage` 값.
- **점검 중 표시:** 하단은 보기 모드(`readOnly`)·**직접 수정** 모드(`setLiveSessionText`로 즉시 `localStorage` 반영, `_lastAccepted` 초기화). **지난 점검 상세**도 `writeSessionRaw`·textarea. **점검 진행 화면에도 "복사" 버튼**을 라벨 행에 두어 권수>0일 때 현재까지 기록을 클립보드에 복사 가능(2026-05-28 추가, 데이터 유실 방지 1차 안전망).

### ④ 시작 전 · 이력 · 내비게이션
- **메인 정리:** 메인 화면 진입 시(표시 전) `removeSessionKeysWithZeroBarcodes()`로 **바코드 0건** 세션 키를 `localStorage`에서 제거. 휴지통 세션은 정리 대상에서 제외(사용자 의도 보존).
- **이전 점검 목록:** `listSessionStorageKeys()`로 접두사 일치 키를 나열(최신순). UI는 활성(휴지통 아님) 세션만 표시. 항목 탭 시 상세에서 **textarea 조회·수정**(`writeSessionRaw`), **휴지통으로 보내기**(확인 후 `softDeleteSession`). 목록 행에서는 **복사 버튼 없음**. 헤더의 "선택"으로 다중 선택 모드 진입 후 "선택한 N개 휴지통으로 보내기"로 일괄 처리.
- **휴지통(2026-05-29 추가, 미리보기 포함):** 지난 점검 목록 하단 "휴지통 (N) →" 버튼으로 진입. 휴지통 화면은 보관 중인 세션 목록 + 항목별 [복구]/[영구 삭제] 인라인 버튼. 항목 상단(아이콘+날짜+권수+chevron 영역)을 탭하면 detail 화면으로 진입해 본문을 미리볼 수 있다. detail은 `selectedIsTrashed`일 때 (a) 상단에 "휴지통에 있는 기록이에요" 안내 박스, (b) [클립보드에 복사하기] 큰 버튼 + [복구]/[영구 삭제] 작은 두 버튼 행, (c) textarea를 readOnly로 강제하고 "휴지통에 있는 기록이라 편집할 수 없어요" 도움말로 분기 렌더. 종료 직후 배너는 노출하지 않는다. 영구 삭제는 `deleteSessionKey`로 본문·메타를 함께 제거하며 별도 확인 대화상자가 뜬다. 복구는 `restoreSession`으로 `deletedAt`만 0으로 초기화(백업 마크는 보존).
- **클립보드:** 백업 경로는 두 군데에서 제공한다.
  - **(1) 점검 진행 중 라벨 행의 "복사" 버튼** — 현재까지 기록만 클립보드로(2026-05-28 추가, 1차 안전망).
  - **(2) 지난 점검 상세의 "클립보드에 복사하기"** — 그 세션 한 건만 클립보드로(원래부터 존재, 2차 안전망 화면; 종료 후 자동 진입 + 안내 배너로 자연스럽게 유도).
  - 목록 행에서는 복사하지 않음(잘못된 항목의 우발적 복사 방지).
- **클라이언트 전용:** 목록/편집은 `typeof window` 이후·마운트 이후에 수행(SSR에서 `localStorage` 직접 접근하지 않음).
- **첫 화면 뒤로가기:** 히스토리 루트에서 뒤로가기 시 **확인 대화상자 없이** 앱 이탈·종료 시퀀스(`scheduleLeaveHostedApp`) 실행.

### ⑤ 카메라 실패 시 (목업 패널)
- **`getUserMedia`/엔진 실패** 등으로 스캔을 열 수 없을 때, 뷰파인더 영역에 **안내 문구 +「카메라 다시 연결하기」** 목업을 표시. (과거 문서에 있던 **데스크톱 가상 숫자 입력** 전용 UI는 현재 코드에 없음.)

## 4. 로드맵·비구현 (Gap / Future)
- **EAN-13 등 체크섬 검증** — 미구현. 현재는 자릿수·숫자 패턴만 사용.
- **서버 동기화·자동 백업** — 미구현. 현재는 사용자가 명시적으로 1·2차 안전망(진행 중 복사 / 종료 후 상세 자동 진입 + 큰 복사 CTA)으로 수동 백업해야 함. 백엔드 도입은 별도 기획 결정 필요.
- **데스크톱·카메라 없는 환경용 가상 스캔 입력** — 미구현(후보).
- **복본(같은 ISBN 2권 이상) 의도적 중복 입력 UX** — 현재는 동일 코드 2초 쿨다운만 있어 의도적 복본 카운트가 어려움. "한 번 더 기록" 액션 또는 카운트 UI 후보.
- **레거시:** 과거 Zustand `persist` 키(`inventory-scanner-scans` 등)는 현재 스캔 파이프라인과 **연동되지 않음**. 필요 시 별도 마이그레이션 스크립트 검토.

## 5. 기술 스택 (Tech Stack)
- **Frontend:** Next.js **16** (App Router), React **19**, Tailwind CSS **4**
- **PWA:** `@ducanh2912/next-pwa`(prod에서 SW 등록, dev 비활성)
- **State:** Zustand **5** (런타임 상태; 스캔 본문 persist 미사용)
- **Storage:** `localStorage` 세션 키 + 헬퍼(`readSessionRaw` / `writeSessionRaw` / 줄 append 패턴)
- **Scanner:** Native `BarcodeDetector` + `@ericblade/quagga2` 폴백

## 6. UI/UX 전략
- **다크 기본:** 현장 가독성·배터리 감안 블랙 톤.
- **한 손 조작:** 하단 고정 **누적 표시·직접 수정 토글·토스트**, 상단 **점검 중단**, 스캔 중 **라이브 인식 패널**.
- **장갑·터치:** 큰 탭 타깃, **장서점검 시작 / 점검 중단** 문구로 의도 명확화.
- **부가 링크:** 스캔 화면에는 외부 SNS 버튼 없음. 안내용 링크는 푸터 등 첫 화면 경로에 한함.
- **헤더 정렬:** 점검 화면도 메인·목록과 동일한 `<main>` flex 패턴으로 정렬을 맞춤. 단 헤더 자체는 홈에만 노출(2026-05-28 결정).
- **오프라인 안내 띠(2026-05-28 확장):** `OnlineStatusBanner`가 헤더와 독립적으로 모든 화면(홈·점검 진행·지난 점검 목록·상세)에서 오프라인일 때만 노출. 비홈 화면에서는 `topmost` prop으로 노치 safe-area를 배너가 직접 흡수해 헤더 부재와 충돌하지 않게 한다. 본 앱은 오프라인에서도 정상 동작하므로 경고 톤이 아닌 안심 톤 문구.
