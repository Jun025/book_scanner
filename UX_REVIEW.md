# 빛나래 장서점검 UX 리뷰 (2026-05-22)

리뷰어: 시니어 프론트엔드/UX 검토.
대상 브랜치 시작 시점: `main` @ `112867f`.

## 0. 사용 맥락 가정

- 사용자: 학교 사서/교사, 학생 도서부 부원(빛나래).
- 기기: 주로 안드로이드/iOS 모바일, 일부 태블릿. 홈 화면에 추가된 PWA 가능성 높음.
- 환경: 서가 사이 좁은 통로, 어두운 조명 가능, 한 손 또는 장갑 끼고 작업 가능. 장시간 반복 스캔.
- 네트워크: 와이파이 불안정해도 동작해야 함(`localStorage` 기반).
- 핵심 동선: `장서점검 시작` → 연속 스캔 → `점검 중단` → 메인의 `지난 점검 기록`에서 상세 진입 → 클립보드 복사 → 메신저 붙여넣기.

## 1. UX 진단

각 항목: [심각도] - 내용 - (근거: 파일:라인)

### A. 연속 스캔 효율성

- **[높음] 화면 꺼짐 방지 (Wake Lock) 미적용.** 장시간 스캔 중 화면이 꺼지면 카메라 트랙이 일시정지·재초기화될 수 있어 마찰이 크다. 모바일 기본 화면 잠금이 30초~1분 수준이라 실제 사용 시 거의 매번 부딪힌다.
  - 근거: `src/components/Scanner.tsx` 전체 — `navigator.wakeLock.request("screen")` 호출 흔적 없음.
- **[중간] 중복 스캔 시 무반응.** `appendDigitScanToActiveSession()` 가 `false` 반환 시(쿨다운 내 같은 코드) 단순 실패 비프만 울린다. 사용자는 "같은 책을 다시 찍었는지", "오류인지" 구분할 단서가 없다.
  - 근거: `src/components/Scanner.tsx:279-282`, `src/store/useScannerStore.ts:152-176`.
- **[중간] 유효하지 않은 코드(비숫자/길이 미달) 인식 시 시각 피드백 부재.** 짧은 비프만 울려, 음소거 환경에서는 인지 불가능.
  - 근거: `src/components/Scanner.tsx:271-278`.
- **[낮음] 손전등(Torch) 토글 없음.** 어두운 서가에서 유용. `MediaTrackCapabilities.torch` 지원 기기 한정.

### B. 오류 상태

- **[양호]** 카메라 권한 실패 시 mock 패널 + `카메라 다시 연결하기` 버튼, visibilitychange 시 자동 재시도 모두 잘 처리됨.
  - 근거: `src/components/Scanner.tsx:310-321, 707-729`.
- **[낮음] 비프 실패 톤이 두 번 짧게 울려서(220Hz 사각파 × 2) 너무 비관적으로 들릴 수 있음.** 의도된 디자인이면 유지.
  - 근거: `src/hooks/useScanBeeps.ts:56-80`.

### C. 모바일 터치 UX

- **[양호]** 주요 버튼 min-h-14(56px) 이상으로 44pt 타깃 기준 충족.
- **[낮음] 디버그 트리거 `i` 버튼의 텍스트가 한 글자(이탤릭 i)이고 `aria-label`이 없다.** 스크린 리더/저시력 사용자에게 의미 전달이 약함.
  - 근거: `src/components/Scanner.tsx:610-619`.

### D. 가독성/접근성

- **[중간] 사용자 폰트 줌 차단.** `viewport: maximumScale=1, userScalable=false`. iOS에서 OS 다이내믹 타입을 무시당하고, 작은 본문(`text-[10px]`, `text-[11px]`)이 학교 학생/교사 환경에서 가독성이 떨어진다.
  - 근거: `src/app/layout.tsx:15-22`, `src/components/Scanner.tsx:760, 782, 831` 등.
- **[중간] `text-zinc-500`(~rgb(113,113,122)) 본문이 검정 배경에서 WCAG AA 미달일 수 있음.** 메인 안내·도움말·라벨 다수.
- **[낮음] 포커스 표시 비일관.** 일부 textarea는 `focus:ring-2`, 대부분 버튼은 포커스 링 없음(active만). 키보드 사용자 어려움.
- **[양호]** 색에만 의존하는 정보는 거의 없음(녹/빨강 + 텍스트 동반).

### E. 진행 상황 가시성

- **[양호]** 헤더에 `총 N권` + 강조 애니메이션, 하단에 누적 textarea, 방금 인식 코드 큰 표시.
- **[낮음]** 전체 대비 진척도는 외부 마스터 DB가 없는 구조라 표시 불가. 현재 누계만으로 충분.

### F. PWA 다움

- **[중간] 오프라인 상태 표시 없음.** `localStorage` 기반이라 사실상 오프라인에서도 동작하지만, 사용자는 "이게 진짜 저장된 건가" 불안할 수 있다. 작은 인디케이터/배너가 안심감을 준다.
- **[낮음] 설치 안내 없음.** iOS Safari는 자동 프롬프트가 없어 별도 안내가 유효. 단, 학교에서 사전 안내했다면 불필요.
- **[양호]** manifest.json, theme color, 서비스워커, apple touch icon 모두 준비됨.

### G. 일관성

- **[양호]** 다크 테마, emerald(액션) / amber(브랜드) / zinc(중립) 톤이 일관됨.
- **[낮음]** 일부 둥근모서리(rounded-2xl vs rounded-3xl) 혼용. 의미 차이가 약함. 보류 가능.
- **[낮음]** 점검 중 화면의 `직접 수정` 버튼이 켜졌을 때(`완료`) 톤은 emerald, 꺼졌을 때 zinc — 좋음. 다만 textarea 자체가 "보기 모드"임을 알아채기는 살짝 약함(테두리/배경이 거의 동일).

### H. 기타

- **[낮음]** 메인 화면의 `장서점검 안내` 카드가 길어 작은 기기에서 `장서점검 시작` 버튼이 첫 화면 아래로 밀릴 수 있음. (실측 필요)
- **[낮음]** 점검 중 비프음을 끄고 싶은 상황(조용한 도서관)이 있을 수 있음 — 음소거 토글이 없음.
- **[낮음]** 목록에서 빠른 삭제 동작 없음. 상세 진입 → 삭제 버튼만 가능. 잘못 시작된 빈 세션은 자동 정리되므로 충분할 수 있음.

## 2. 작업 범위 결정

순수 UX/UI 개선만, 회귀 위험이 낮은 항목을 골라 구현한다.

### 적용 (이 PR에서 함)

1. **[A·높음] Wake Lock**: 스캔 모드 + 카메라 동작 중 화면 꺼짐 방지. 페이지가 가려지면 release, 다시 보이면 재요청. 미지원 브라우저는 조용히 무시.
2. **[A·중간] 중복 스캔 시각 피드백**: `appendDigitScanToActiveSession`가 거절(중복)했을 때 토스트로 "이미 기록한 번호" 안내 + 부드러운 비프 한 번. (실패 비프 두 번은 비숫자 케이스에만 유지)
3. **[A·중간] 비숫자 코드 인식 시 토스트**: 짧은 안내("숫자가 아니라 넘어갔어요"). 쿨다운 공유.
4. **[D·낮음] `i` 버튼 aria-label** 추가, 디버그 트리거임을 명확화.
5. **[D·낮음] 본문 색 대비 보강**: 가장 작은 보조 텍스트의 `text-zinc-500` → `text-zinc-400` 또는 `text-zinc-300/80` 으로 한 단계 밝게(중요 라벨만, 디자인 일관성 유지 범위에서).
6. **[F·중간] 오프라인 인디케이터**: 헤더 아래 얇은 띠로 `navigator.onLine` 변화 표시. 단, 본 앱은 오프라인에서도 정상 동작하므로 경고가 아닌 안내 톤.
7. **[H·낮음] 비프 음소거 토글**: 디버그 모달에 토글 1개. `localStorage`에 boolean 1개 저장. 진동은 별도 컨트롤로 늘리지 않음(시각·청각 피드백이 다중이라 가장 쉽게 꺼지는 청각만 컨트롤).
8. **[G·낮음] 보기 모드 textarea 시각 보강**: 보기 모드일 때 배경/테두리 톤을 살짝 다르게 해 "편집 가능 여부"를 직관화. 색 변경만, 레이아웃·동작 변경 없음.
9. **[H·낮음] 메인 안내 카드 압축**: 안내 카드 패딩·문구 줄여 첫 화면에서 `장서점검 시작` 버튼이 잘 보이도록. 문구 의미는 보존.

### 보류 — 사람 확인 필요

- **userScalable=false 해제**: 줌 허용은 접근성에 분명히 좋지만, 카메라+textarea 동시 화면에서 사용자 핀치줌이 의도치 않게 트리거되어 스캔 마찰을 키울 위험이 있음. 의도된 잠금일 가능성이 높아 디자인 결정 확인 필요.
- **손전등(Torch) 토글**: API 호환성·아이콘 추가·동작 검증이 필요해 회귀 위험이 있음. 학교 도서관 조명 상태 확인 후 별도 PR 권장.
- **iOS 홈 화면 추가 안내 모달**: 학교에서 별도 안내했다면 사족. 비즈니스 결정 필요.
- **목록에서 빠른 삭제(스와이프)**: 새 인터랙션 패턴. 빈 세션은 자동 정리되므로 우선순위 낮음. 사용자 데이터 손실 위험.
- **`text-zinc-500` → 대비 일괄 상향**: 위 5번에서 라벨 위주로만 부분 적용. 전수 변경은 디자인 톤이 바뀌므로 보류.
- **EAN-13 체크섬 검증**: PRD에서 로드맵으로 명시. UX 범위 밖.

- **카메라 손전등(Torch) 토글**: 기기별 capability 호환·아이콘 추가가 따라붙고 본 PR 회귀 위험이 있어 별도 PR 권장.
- **iOS 홈 화면 추가 안내 모달**: 학교 사전 안내 여부에 따라 사족이 될 수 있음 — 비즈니스 결정 필요.
- **목록 항목 빠른 삭제(스와이프)**: 새 인터랙션 패턴 + 데이터 손실 위험. 빈 세션은 자동 정리되므로 우선순위 낮음.
- **`text-zinc-500`의 전수 상향**: 디자인 톤이 바뀌므로 라벨 위주 부분 적용에 그침.

### 3.4 실기기 테스트 권고 (출근 후 확인)

내가 직접 실기기에서 시연하지 못한 부분이라 실제 도서관 환경에서 짧게 한 번 봐주세요:

1. **Wake Lock 동작**: 스캔 시작 후 1~2분 그대로 두었을 때 화면이 꺼지지 않는지(Android Chrome / iOS Safari 16.4+). 미지원 기기에서도 기존 동작이 유지되는지.
2. **중복 토스트 톤**: 같은 책을 빠르게 두 번 비췄을 때 amber 톤 토스트가 부드럽게 표시되는지, 너무 잦지 않은지.
3. **음소거 토글 영속성**: 모달에서 끄고 앱을 닫았다가 다시 열었을 때 꺼진 상태가 유지되는지(localStorage 확인).
4. **오프라인 띠**: Wi-Fi/모바일 데이터 끄면 띠가 즉시 표시되고, 다시 켜면 즉시 사라지는지.
5. **보기/편집 모드 구분**: 점검 중 textarea의 dashed border가 한눈에 "지금은 자동 누적"임을 전달하는지. 학생 도우미의 첫 인상을 한 번 봐주세요.
6. **메인 카드 압축 후 시작 버튼 가시성**: 가장 작은 학교 기기(예: iPhone SE급)에서 `장서점검 시작`이 첫 화면(스크롤 없이)에 보이는지.
