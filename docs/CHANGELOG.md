## 2026-05-29

### UX 헌법 명문화 + 다중 선택 예외 가드
- `.cursorrules` 최상단에 UX 헌법 한 줄을 명문화: "사용자는 어린 학생이다. 화면당 하나의 명확한 행동만 둔다. 여러 개를 모으거나 고르게 하지 않는다. 한 세션 = 한 번의 점검 = 한 번 복붙이 핵심 멘탈 모델이다."
- 이미 구현된 "지난 점검 목록 다중 선택 + 휴지통 일괄 이동"이 헌법과 일부 충돌하는 점을 정직하게 짚고, **예외 동선(정리/관리 작업에 한정)**으로 명문화. 일상 점검 동선(시작·진행·종료·복사)에는 절대 들이지 않는 가드레일을 헌법 인용 블록에 박아 둠.

### 데이터 무결성 — 시계 점프 충돌 방어
- 세션 키가 `book-scanner:session:` + ISO8601 시작 시각인데, OS 시각이 되감기면 동일 ISO가 다시 만들어져 `writeSessionRaw(key, "")`가 기존 점검 본문을 덮어쓸 위험이 있었다. **P1 데이터 유실 가능성.**
- `makeSessionStorageKey`가 충돌 시 `-2`, `-3`, ... suffix를 붙여 첫 빈 슬롯을 사용하도록 강화. 999회 충돌은 비현실적이지만 무한루프 방지로 `-<ms>-<random4>` 최종 안전망.
- `formatSessionLabel`은 suffix를 떼고 ISO만 파싱해 사용자에겐 동일 시각 라벨로 보임(같은 base의 카드가 두 개 보일 수 있지만 카드 형태는 동일 — 헌법 "화면당 하나" 정신은 깨지 않음).
- 정렬은 문자열 비교에서 `Z < Z-` 덕에 suffix 키가 자연스럽게 더 최신으로 노출. 메타 매핑(`metaKeyForSessionKey`)도 prefix만 교체하므로 suffix 호환. 충돌이 없는 일반 경로에서는 종전 키 형식 그대로 — **마이그레이션 불필요.**

### Vitest 단위 테스트 도입
- `vitest` 4.1.7 + `jsdom` 29.1.1 devDependency 설치, `pnpm test` 스크립트 추가.
- 3 파일·53 테스트로 핵심 invariant 회귀 보호: `mergeSessionMetaRaw` 부분 갱신, 메타 하위 호환, `removeSessionKeysWithZeroBarcodes` 분기(권수>0 보존·휴지통 보존·orphan 정리), 키 충돌 방어, 동일 코드 쿨다운, 텍스트 정규화 경계.
- React 컴포넌트·카메라·실제 OS 시계 조작은 본 범위 제외(별도 도구 도입 시점에).
- CI(GitHub Actions) 미도입 — main 직접 push 흐름이라 pre-merge 게이트 자리 없음. 로컬 push-전 tsc/lint/build/test 4종 통과 게이트로 운영.

### 휴지통 미리보기
- 휴지통 항목 상단 영역을 탭하면 detail 화면을 휴지통 컨텍스트로 재사용해 본문 미리보기 + 클립보드 복사 + 복구/영구 삭제 가능. 영구 삭제 직전 안전 확인 동선.
- detail은 `selectedIsTrashed`일 때 (a) "휴지통에 있는 기록이에요" 회색 안내 박스, (b) [복사하기] + [복구]/[영구 삭제] 버튼 행, (c) textarea readOnly 강제로 분기 렌더.

## 2026-05-28

### 휴지통(2단계 삭제) 도입
- "삭제"가 즉시 영구가 아니라 휴지통 이동(소프트 삭제)으로 변경. 본문은 보존되며 사용자가 휴지통에서 명시적으로 [복구] 또는 [영구 삭제]를 선택해야 영구 삭제.
- 메타 스키마에 `deletedAt` 필드 추가. `mergeSessionMetaRaw` 부분 갱신 헬퍼로 `backedUpAt`/`deletedAt`이 서로 덮어쓰지 않게 함. 두 필드 모두 0이 되면 메타 키 자체 삭제(orphan 사전 방지).
- 지난 점검 목록 헤더에 **"선택"** 버튼으로 다중 선택 모드 진입 → 하단 "선택한 N개 휴지통으로 보내기"로 일괄 처리. (이후 헌법 예외 동선으로 명문화)
- 휴지통 화면: 항목별 [복구]/[영구 삭제] 인라인 버튼 + "전체 영구 삭제" 일괄은 의도적으로 제공 안 함(실수 방지).

### 백업 추적 메타 + 뱃지
- 본문과 분리된 `book-scanner:meta:` 네임스페이스에 `{ backedUpAt }` JSON 보관. 진행 중 복사·상세 복사 실제 성공 시 자동 마크.
- 지난 점검 목록의 권수>0 세션에 뱃지로 시각화 — 초기에 "보냄"/"아직 안 보냄"으로 도입했다가 학생을 책망하는 톤이라는 피드백을 받아 **"✓ 복사한 적 있음"**(목련 그린, 완료) / **"● 신규"**(연꽃 로즈, 강조)으로 톤 갱신.
- 종료 직후 자동 진입 배너는 이미 복사된 적 있는 세션에는 노출하지 않아 중복 안내 제거.

### 데이터 유실 방지 — 진행 중 복사 + 종료 후 상세 자동 진입
- 점검 진행 화면 하단 라벨 행에 [복사] 버튼 신설(권수>0일 때). 점검을 중간에 사서께 보낼 수 있는 안전망.
- 권수>0으로 종료 시 메인 대신 그 세션의 지난 점검 상세로 자동 이동 + "사서께 전달하려면 복사하세요" 안내 배너. 비차단형(모달 아님)이라 "확인 없이 즉시 종료" 원칙과 양립.
- 한때 "전체 N개 한 번에 보내기"(Web Share → 클립보드 → .txt 다운로드 폴백)를 3차 안전망으로 도입했으나, **불필요하다는 판단으로 같은 회차 안에 제거**. exportSessions.ts 라이브러리 통째로 삭제.

### 오프라인 안내 띠 전 화면 노출
- `OnlineStatusBanner`를 홈에만 노출하던 것을 모든 화면(점검 진행 포함)으로 확장. 비홈 화면에서는 `topmost` prop으로 노치 safe-area를 배너가 직접 흡수.
- 본 앱은 오프라인에서도 정상 동작하므로 경고가 아닌 안심 톤("오프라인이에요. 기록은 이 기기에 그대로 저장돼요").

### 비프 시스템 제거
- 도서관 정숙 환경 부담을 이유로 Web Audio 비프 시스템(`useScanBeeps` 훅, sound-muted 토글)을 통째로 제거. 인지 피드백은 진동 + 녹색 플래시 + 토스트만으로 충분하다는 결정.

### 헤더·info·종료 버튼 재배치
- 헤더("도서부 빛나래 / 빛나래 장서점검")는 홈 화면에만 노출하도록 변경(점검 진행·지난 점검 목록/상세에서는 카메라·콘텐츠 영역을 위해 헤더 제거).
- 기기 정보(`i`) 버튼을 헤더 우측에서 푸터 인스타 옆으로 이동(`DebugInfoButton`).
- 점검 종료 동작을 헤더의 "점검 중단" 텍스트 버튼에서 진행 화면 상단 "지금까지 점검" 카드 왼쪽의 셰브런-백 아이콘으로 이동. 권수에 따라 aria-label만 분기.

### 디자인 토큰 — 연꽃 로즈 · 목련 그린 팔레트
- 가람고 교화(연꽃)·교목(목련)을 모티프로 한 디자인 토큰 도입. `--brand` = lotus-600, `--accent` = magnolia-600. 라이트/다크 양쪽 WCAG AA 대비 보장.
- Pretendard Variable 폰트 적용(jsDelivr CDN).
- Toss풍의 절제된 카피·간격·`.press` 마이크로 인터랙션.

### 안내 카피 다듬기
- 홈 안내 카드 3번째 항목 "Wi-Fi가 끊겨도 찍은 순간부터 저장돼요" → "중간에 인터넷이 끊겨도 점검 기록이 저장돼요" — 모바일 데이터·Wi-Fi 구분 없는 일반 표현으로.

### 빌드 가드 강화
- 프로덕션 빌드가 `--webpack` 없이 시작되면 `next.config.ts`가 즉시 throw하는 가드 추가. `@ducanh2912/next-pwa`는 webpack 전용이라 Next.js 16의 기본 Turbopack으로 빌드하면 서비스 워커가 누락된다.
- README의 Cloudflare 배포 절에 동일 취지의 경고 박스 추가.
